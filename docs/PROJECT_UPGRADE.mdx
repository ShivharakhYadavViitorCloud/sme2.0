## Environment variables

### Docusaurus (build / `.env.local`)

| Variable | Purpose |
|----------|---------|
| `GTAG_ID` | Optional Google Analytics measurement ID |
| `PUBLIC_API_BASE_URL` | Base URL of the SME API server **without** a path suffix, e.g. `http://localhost:8787` or `https://api.yourdomain.com`. The browser calls `POST {PUBLIC_API_BASE_URL}/api/chat` and `POST .../api/feedback`. |

### Node API (`server/` ‚Äî see `server/.env.example`)

| Variable | Purpose |
|----------|---------|
| `MONGODB_URI` | MongoDB connection string (feedback + chat audit) |
| `OPENAI_API_KEY` | OpenAI key for `/api/chat` |
| `OPENAI_MODEL` | Optional model id (default `gpt-4o-mini`) |
| `PORT` | Listen port (default `8787`) |
| `CORS_ORIGINS` | Comma-separated allowed browser origins (must include your Docusaurus dev server and production site origin) |

Copy the repo root `.env.example` to `.env.local` for Docusaurus, and `server/.env.example` to `server/.env` for the API (never commit secrets).

## Versioning workflow

1. Edit files under `docs/` (the **Next** docs).
2. When you want to freeze a release: `npm run docs:version -- X.Y` (e.g. `1.1`).
3. Commit `versioned_docs/`, `versioned_sidebars/`, and `versions.json`.

Older versions live under `/docs/X.Y/‚Ä¶`; the latest unreleased docs are under `/docs/next/‚Ä¶` when **Next** is configured as the current label.

## SME API server (`server/`)

The **Express + MongoDB** service in `server/` exposes:

- `POST /api/chat` ‚Äî OpenAI proxy (stores a short audit row in MongoDB)
- `POST /api/feedback` ‚Äî Persists doc feedback (`path`, `title`, `helpful`, ‚Ä¶)
- `GET /health` ‚Äî Liveness

Run locally: from the repo root, `npm run api:dev` (requires `server/.env`). Point `PUBLIC_API_BASE_URL` at the same origin (e.g. `http://localhost:8787`) when running `npm start` for the docs site.

GitHub Pages only serves static files; host the API on a VM, container platform, or PaaS, set `PUBLIC_API_BASE_URL` in the **build** environment for production docs, and ensure `CORS_ORIGINS` includes your GitHub Pages URL.
